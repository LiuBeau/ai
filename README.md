# 🧠 LifeMate — 个人生活智能助手

> 一个从零构建的、具备长期记忆、自主规划与知识检索能力的桌面 AI Agent

LifeMate 是一个运行在 Windows 桌面上的个人生活助手，它不仅仅是一个“聊天机器人”，而是真正具备**自主思考、工具调用、记忆保持和自我反思**能力的智能体（Agent）。

## ✨ 核心特性

- 🧠 **自主规划**：基于 ReAct 模式，Agent 能自主拆解任务、调用工具、观察结果并迭代执行。
- 📝 **长期记忆**：通过 SQLite + ChromaDB 构建两层记忆系统，既记住你的偏好，也记住你的生活碎片。
- 🔧 **工具生态**：天气查询、日程管理、桌面通知、知识检索，并支持灵活扩展。
- 💬 **桌面交互**：基于 CustomTkinter 构建的现代化聊天界面，常驻桌面，随时响应。
- ⏰ **主动服务**：支持定时推送，每天早上自动推送天气与生活建议。
- 🧩 **RAG 知识库**：内置生活常识向量库，支持“穿衣搭配”、“旅游出行”等生活话题的语义检索。
- 🌀 **自我反思**：Agent 能识别失败调用并主动纠错，避免无效循环。

## 🧠 核心设计思想

### 1. ReAct 自主规划
Agent 不是被动响应，而是主动思考：`思考 → 行动 → 观察 → 再思考 → 完成`。每一步都基于前一步的结果动态决策，实现了真正的自主性。

### 2. 双层记忆系统
- **SQLite 层**：存储结构化偏好（如“怕冷”、“常用城市”），跨会话保持。
- **ChromaDB 层**：存储非结构化生活碎片（如“我喜欢吃火锅”），支持语义检索。

### 3. 工具注册表模式
所有工具统一注册在 `TOOLS` 字典中，新增工具无需修改核心逻辑，实现“开闭原则”。

### 4. 自我反思机制
当工具调用失败时，Agent 会主动分析原因并调整策略，避免重复无效操作。

🤖 关于“真智能体”的思考
本项目在演进过程中触及了当前 AI Agent 领域的核心矛盾：“工具扩展”与“自主智能”之间的张力。

现状：每新增一个能力就需要新增一个工具定义，本质仍是“设计”而非“涌现”。

方向：真正的智能体应具备 工具发现、工具生成 和 自我进化 能力。

本项目已在 L6 版本中探索了“通用知识检索”方向，将知识注入向量库，让 Agent 通过语义检索实现新能力，而无需新增专用工具。
